# kakebo

家計簿管理アプリケーション。技術スタックは未定（今後の開発方針に応じて決定する）。

## 現在の状態

- プロジェクトは初期段階で、ソースコードはまだ存在しない。
- ローカルにgitリポジトリは未初期化、GitHub上のリモートリポジトリも未作成。
- 本ファイルのGit運用ルールを適用する前に、`git init` とGitHubリモートリポジトリの作成・紐付けが必要。

## Git運用ルール

- **コードに変更（Edit/Write等によるファイル編集）を行うたびに、その都度コミットしてGitHubにプッシュする。** 作業をまとめて後でコミットするのではなく、1回の変更単位ごとにコミット&プッシュを行う。
- コミットメッセージは変更内容を簡潔に要約する（日本語・英語どちらでも可、内容が分かることを優先する）。
- プッシュ前に `git status` / `git diff` で変更内容を確認する。
- force push、`git reset --hard` などの破壊的操作は明示的な指示がない限り行わない。
- 機密情報（APIキー、認証情報など）を含むファイルはコミットしない。
